# ESP32 Intercom

Полнодуплексный Wi-Fi интерком на базе ESP32 с I2S микрофоном и усилителем MAX98357A.

## 🎯 Возможности

- **Полнодуплексная аудиосвязь** — одновременная передача и приём, как у телефона
- **Режим Push-to-Talk** — удержание кнопки для передачи (полудуплекс)
- **Web-интерфейс настройки** — удобная конфигурация с телефона/ПК
- **Captive Portal** — автоматическое появление страницы настройки при первом запуске
- **Кросс-подсетевая связь** — поддержка связи между устройствами в разных подсетях
- **Качественное аудио** — 16-бит PCM, до 44100 Гц, настраиваемое усиление
- **Связь по UDP** — минимальная задержка, без промежуточных серверов

## 📦 Компоненты

| Компонент | Описание |
|-----------|----------|
| ESP32 (DevKit v1) | Микроконтроллер с Wi-Fi |
| SPH0645LM4H-B | I2S MEMS микрофон (аналог "43434") |
| MAX98357A | 3W Class D I2S усилитель |
| Динамик 4-8 Ом | 3W или более |
| Кнопка (тактовая) | Для управления вызовами |
| LED | Индикатор состояния |
| Резистор 220 Ом | Для LED |

## 📐 Схема подключения

```
                    ┌─────────────────┐
                    │      ESP32      │
                    │                 │
  Микрофон SPH0645  │                 │  MAX98357A
  ┌──────────┐      │                 │  ┌──────────┐
  │ VDD ─────┼──────┤ 3V3            │  ├── VDD    │
  │ GND ─────┼──────┤ GND            │  ├── GND    │
  │ BCLK─────┼──────┤ GPIO14         │  ├── BCLK───┤ GPIO26
  │ LRCLK────┼──────┤ GPIO15         │  ├── LRC────┤ GPIO27
  │ DOUT─────┼──────┤ GPIO32         │  ├── DIN────┤ GPIO25
  │ L/R ─────┼──────┤ 3V3            │  └── SDMODE─┤ GND (или 3V3)
  └──────────┘      │                 │  ┌──────────┐
                    │                 │  │ Динамик  │
                    │                 │  ├─ + ──────┤ OUT+ MAX98357A
                    │                 │  └─ - ──────┤ OUT- MAX98357A
                    │                 │  └──────────┘
  Кнопка            │                 │
  ┌──┐              │                 │
  │  ├──────────────┤ GPIO4          │
  └──┘              │                 │
  GND ──────────────┤ GND            │
                    │                 │
  LED               │                 │
  LED ──[220Ω]──────┤ GPIO2          │
  GND ──────────────┤ GND            │
                    └─────────────────┘
```

### Пины по умолчанию

| Функция | GPIO |
|---------|------|
| I2S Mic BCLK | GPIO14 |
| I2S Mic LRCLK | GPIO15 |
| I2S Mic Data In | GPIO32 |
| I2S SPK BCLK | GPIO26 |
| I2S SPK LRCLK | GPIO27 |
| I2S SPK Data Out | GPIO25 |
| Кнопка | GPIO4 (с внутренней подтяжкой) |
| LED | GPIO2 |

> Все пины можно изменить в файле `src/pins.h`.

## 🏗️ Сборка и прошивка

### Требования

1. [PlatformIO](https://platformio.org/) (рекомендуется) или Arduino IDE
2. ESP32 DevKit v1 (или совместимая плата)

### Установка через PlatformIO

```bash
# Клонирование репозитория
git clone https://github.com/ShtirlizVD/Intercom_ESP32.git
cd Intercom_ESP32

# Сборка
pio run

# Прошивка
pio run -t upload

# Загрузка файловой системы (веб-интерфейс)
pio run -t uploadfs

# Монитор порта
pio device monitor
```

### Через Arduino IDE

1. Установите поддержку ESP32 в Arduino IDE
2. Установите библиотеку ArduinoJson (v7+)
3. Откройте файл `src/main.cpp`
4. Добавьте все `.cpp` и `.h` файлы в проект
5. Выберите плату "ESP32 Dev Module"
6. Скомпилируйте и прошейте

## ⚙️ Настройка

### Первоначальная настройка

1. Подключите питание — ESP32 запустится в режиме **точки доступа (AP)**
2. На телефоне/ПК найдите Wi-Fi сеть `Intercom_ESP32-XXXX`
3. Подключитесь с паролем `12345678`
4. Откройте в браузере `http://192.168.4.1`
5. Настройте подключение к вашему Wi-Fi
6. Сохраните — устройство перезагрузится и подключится к сети

### Настройка двух устройств

Для каждого устройства:

1. Подключите к своему Wi-Fi через веб-интерфейс
2. В разделе **"📞 Интерком"** укажите IP-адрес второго устройства
3. Нажмите **"💾 Сохранить настройки"**

## 🌐 Сетевая конфигурация (кросс-подсетевая связь)

Если устройства находятся в разных подсетях (например, дом `192.168.1.x` и гараж `192.168.2.x`), нужно правильно настроить маршрутизацию между ними.

### Вариант 1: Режим моста (рекомендуется) ✅

Самый простой и надёжный способ. Гаражный роутер работает как беспроводной мост/коммутатор без NAT.

**Настройка гаражного роутера:**

1. Откройте настройки роутера
2. Найдите опцию **"Режим моста"** (Bridge Mode), **"WDS"** или **"Точка доступа"** (Access Point)
3. Отключите DHCP-сервер на гаражном роутере
4. Подключите гаражный роутер к радиомосту (LAN-порт к LAN-порту, или через WDS)
5. **Результат:** оба ESP32 будут в одной подсети `192.168.1.x`

> В этом случае устройства видят друг друга напрямую — никаких дополнительных настроек не нужно.

### Вариант 2: Статические маршруты

Если режим моста недоступен, добавьте статические маршруты на обоих роутерах.

**Схема сети:**

```
[Домашний роутер]          [Радиомост]          [Гаражный роутер]
192.168.1.1          ———    192.168.1.x    ———    192.168.2.1
                     ———    192.168.2.y    ———
```

**На домашнем роутере (192.168.1.1):**
- Добавьте статический маршрут:
  - Сеть назначения: `192.168.2.0`
  - Маска: `255.255.255.0`
  - Шлюз: `<IP радиомоста в сети 192.168.1.x>`

**На гаражном роутере (192.168.2.1):**
- Добавьте статический маршрут:
  - Сеть назначения: `192.168.1.0`
  - Маска: `255.255.255.0`
  - Шлюз: `<IP радиомоста в сети 192.168.2.x>`
- Отключите NAT (если возможно) для локального трафика между подсетями

**На радиомосте (если это отдельное устройство с настройками):**
- Убедитесь, что IP-форвардинг включён
- Он должен иметь IP-адрес в обеих подсетях

### Вариант 3: VPN (WireGuard)

Для наиболее сложных сетевых конфигураций можно использовать WireGuard VPN:

1. Установите WireGuard на домашнем и гаражном роутерах (или на отдельных мини-ПК)
2. Создайте туннель между ними
3. ESP32 устройства будут видеть друг друга через VPN-туннель

### Проверка связи

После настройки маршрутизации проверьте связь с компьютера в домашней сети:

```bash
ping 192.168.2.x  # IP ESP32 в гараже
```

Если пинг проходит — интерком будет работать. Если нет — проверьте настройки маршрутов и файрволы.

## 📞 Использование

### Телефонный режим (полнодуплекс)

1. Нажмите кнопку на ESP32 — начнётся вызов удалённого устройства
2. На удалённом устройстве загорится/заморгает LED
3. Нажмите кнопку на удалённом устройстве — разговор начнётся
4. Оба собеседника могут говорить одновременно
5. Нажмите кнопку на любом устройстве — разговор завершён

### Режим Push-to-Talk (PTT)

1. Нажмите кнопку — вызов
2. Нажмите на втором устройстве — ответ
3. **Удерживайте кнопку** для передачи речи
4. Отпустите — микрофон замолкает, но вы продолжаете слышать собеседника
5. Для завершения — нажмите кнопку на любом устройстве (коротко, не удерживая)

### LED индикация

| Состояние LED | Значение |
|--------------|----------|
| Выключен | Ожидание (нет вызова) |
| Мигает быстро (300 мс) | Исходящий вызов |
| Мигает медленно (500 мс) | Входящий вызов |
| Горит постоянно | Разговор |

## 📡 Протокол

### Управляющие сообщения (UDP, порт 8080)

Формат: JSON

```json
{"t": "ring", "f": "DeviceName"}
{"t": "answer", "f": "DeviceName"}
{"t": "hangup", "f": "DeviceName"}
{"t": "ping", "f": "DeviceName"}
{"t": "pong", "f": "DeviceName"}
{"t": "busy", "f": "DeviceName"}
```

### Аудио поток (UDP, порт 8081)

Формат пакета:

```
[SeqNum: 2 байта][Timestamp: 4 байта][PCM данные: N байт]
```

- **PCM**: 16-бит signed, mono
- **Частота**: настраивается (8000 / 16000 / 22050 / 44100 Гц)
- **Размер кадра**: 20 мс

### Потребление полосы пропускания

| Частота | Битрейт | Размер кадра |
|---------|---------|--------------|
| 8000 Hz | 128 kbps | 320 байт |
| 16000 Hz | 256 kbps | 640 байт |
| 22050 Hz | 353 kbps | 882 байт |
| 44100 Hz | 706 kbps | 1764 байт |

## 🔧 API (HTTP)

Веб-интерфейс доступен на порту 80. Поддерживаются REST API:

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/status` | Статус устройства |
| GET | `/api/config` | Текущая конфигурация |
| POST | `/api/config` | Обновить конфигурацию |
| POST | `/api/wifi/scan` | Сканировать Wi-Fi сети |
| POST | `/api/wifi/connect` | Подключиться к Wi-Fi |
| POST | `/api/wifi/disconnect` | Отключиться от Wi-Fi |
| POST | `/api/reboot` | Перезагрузить ESP32 |
| POST | `/api/factory-reset` | Сброс настроек |

## 🐛 Решение проблем

### Нет звука
- Проверьте подключение I2S пинов (BCLK, LRCLK, Data)
- Для MAX98357A: убедитесь, что SD_MODE подключён (к GND или 3V3)
- Увеличьте усиление микрофона и громкость в веб-интерфейсе
- Проверьте питание — ESP32 + усилитель требуют хороший источник питания (2A+)

### Не подключается к Wi-Fi
- Проверьте SSID и пароль
- ESP32 поддерживает только 2.4 Гц сети
- Попробуйте перезагрузить роутер

### Устройства не видят друг друга
- Убедитесь, что оба устройства подключены к сети
- Проверьте, что IP-адреса указаны правильно
- Пропингуйте одно устройство с другого (если ESP32 настроен как клиент, можно пропинговать с компьютера)
- Для разных подсетей — проверьте статические маршруты на роутерах

### Прерывистый звук
- Уменьшите частоту дискретизации (до 16000 или 8000 Гц)
- Проверьте качество Wi-Fi сигнала (RSSI должен быть лучше -70 дБм)
- Убедитесь, что радиомост обеспечивает достаточную пропускную способность

### Задержка звука
- Задержка обычно 20-50 мс в локальной сети — это нормально
- Для уменьшения задержки используйте 8000 Гц
